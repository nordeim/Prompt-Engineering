## 3. Project Philosophy & Community

### 3.1 Some of My Humble Opinions

- I recently discovered that the Nature Skills I designed has been noticed and referenced by Google DeepMind. They drew inspiration from its citation system, script ideas, and skill design philosophy to launch Science Skills. Honestly, this makes me quite gratified—when top international AI institutions begin drawing inspiration from our work, it shows that original ideas from Chinese developers are being seen by the world. This is not the frustration of being copied, but rather the natural outward growth of influence after Chinese strength takes root in open-source soil.
- The focus of our Skills design has never been about requiring everyone to thoroughly digest this philosophy; rather, the philosophy itself is designed to be understood and reused by machines. If you want to create a brand-new Skill or adapt it to your own specialized domain, you don't need to start from scratch—simply send the Nature Skills GitHub address to Codex, and it will automatically learn the design essence within, helping you complete the creation and modification of new Skills. This is the true liberation of ideas: they no longer depend on word-of-mouth transmission, but flow directly through AI into every corner that needs them.
- The true value of Nature Skills may not lie in those specific skill modules, but in the new door it quietly opens—it makes many people realize for the first time that they can use Codex or intelligent agents to control their local computer for research. I have been fortunate to witness and accompany many people through the transformation of their research paradigm. The moment they exclaim "So this is how research can be done," this cognitive breakthrough and liberation is far more meaningful to me than the Skills themselves. It is not the success of a tool, but a new way of thinking beginning to spread among people.
- In the present day, almost all practical tools can be distilled into standardized processes, and standardized processes happen to be exactly what can be encapsulated into reusable skills.

## 4. Quick Start

After installation, you can directly hand papers, paragraphs, review comments, or task descriptions to the Agent. The following prompts can be copied and used directly:

| What You Want To Do | Just Say This |
|---|---|
| Read paper / Chinese-English对照 | `Turn this PDF into a figure-text-aligned Chinese-English对照 Markdown reader.` |
| Generate literature report PPT | `Turn this paper into a Chinese group meeting report PPT, keeping key figures and source annotations.` |
| Polish or translate paper paragraphs | `Rewrite this Chinese into Nature-style English, preserving academic meaning.` |
| Write abstract, introduction, or discussion | `Based on these results and figures, help me draft a Nature-style abstract and introduction.` |
| Pre-submission review simulation | `Evaluate this manuscript from a Nature reviewer perspective, providing three reviewer reports.` |
| Respond to review comments | `Based on this revision email, help me write a point-by-point response, cover letter, and mark where the revised manuscript needs to be highlighted in red.` |
| Search literature, citations, and citation profiles | `Compile this article's citation count, strict external citation count, DOI, and check whether any of the citer's are academicians, Fellows, or domain leaders.` |
| Create research figures or paper diagrams | `Based on this method and results, help me generate publication-grade research figures or paper diagram drafts.` |

If you're unsure which skill to use, simply describe the task; if you already know the skill name, you can explicitly write "use `nature-reader`" or "use `nature-response`" in the prompt.

## 5. Installation

`nature-skills` is a set of reusable skill packages organized around `SKILL.md`. Each top-level skill directory under `skills/` is an installable unit, e.g., `nature-*`; `nature-shared` is a shared support package for other skills to read, and is not counted as an independently triggerable skill in the skill index by default.

### 5.1 `npx skills` Installation Method

You need to install [Node.js 18 or higher](https://nodejs.org/) first. No global CLI installation is required; first check the installable skill names in the repository:

```bash
npx skills add Yuan1z0825/nature-skills --list
```

Install all skills globally to Codex. `nature-shared` will be included with the full installation, so skills that depend on shared reference materials will work properly:

```bash
npx skills add Yuan1z0825/nature-skills --global --agent codex --skill '*' --yes --copy
```

To install a single skill for the current project only, omit `--global`. For example:

```bash
npx skills add Yuan1z0825/nature-skills --agent codex --skill nature-figure --yes --copy
```

When installing `nature-reader`, `nature-paper2ppt`, `nature-polishing`, or `nature-writing` individually, also select the shared support package:

```bash
npx skills add Yuan1z0825/nature-skills --global --agent codex \
  --skill nature-reader --skill nature-shared --yes --copy
```

You can also install all skills to all agents supported by the CLI:

```bash
npx skills add Yuan1z0825/nature-skills --all
```

Check global installation results and update:

```bash
npx skills list --global --agent codex --json
npx skills update --global --yes
```

Update only one skill, or only skills in the current project:

```bash
npx skills update nature-reader --global --yes
npx skills update --project --yes
```

Skill selection parameters use the frontmatter names displayed by `--list`; for example, directory `nature-proposal-writer` currently displays as `researchwrite`. `npx skills` manages skill files; optional runtime dependencies such as Python, R, browser, or MCP still need to be configured separately as described below.

### 5.2 Claude Code Installation Method

Claude Code cannot directly use `scripts/update-codex-skills.sh`, because this script only synchronizes to Codex's `~/.codex/skills/`. When used for Claude Code, it is recommended to keep a stable local clone, then use a subagent or slash command to point to the real `skills/*/SKILL.md`. This does not break the skill directory structure, and can continue to read `references/`, `static/`, `manifest.yaml`, scripts, assets, and `skills/nature-shared/`.

If you haven't installed Claude Code yet:

```bash
npm install -g @anthropic-ai/claude-code
claude
```

First clone the repository to a stable path:

```bash
mkdir -p ~/ai-skills
cd ~/ai-skills
git clone https://github.com/Yuan1z0825/nature-skills.git
```

Recommended method: create Claude Code subagent wrappers for commonly used skills. Using `nature-reader` as an example:

```bash
mkdir -p ~/.claude/agents
cat > ~/.claude/agents/nature-reader.md <<'EOF'
---
name: nature-reader
description: Use for Chinese-English paper reading, figure-aware translation, and source-grounded paper notes.
---

When invoked, first read `~/ai-skills/nature-skills/skills/nature-reader/SKILL.md` and follow it as the governing workflow.
Read supporting files from `~/ai-skills/nature-skills/skills/nature-reader/` and `~/ai-skills/nature-skills/skills/nature-shared/` only when needed.
Do not replace this skill with a generic paper-reading response.
EOF
```

Then start a new Claude Code session and directly request to use this subagent:

```text
Use the nature-reader subagent to turn this paper into a Chinese-English Markdown reader.
```

If you prefer slash commands, you can also create command wrappers:

```bash
mkdir -p ~/.claude/commands
cat > ~/.claude/commands/nature-reader.md <<'EOF'
Read `~/ai-skills/nature-skills/skills/nature-reader/SKILL.md` first and follow it strictly.
Read directly needed supporting files from `~/ai-skills/nature-skills/skills/nature-reader/` and `~/ai-skills/nature-skills/skills/nature-shared/`.

$ARGUMENTS
EOF
```

Use in Claude Code:

```text
/nature-reader Turn this paper into a complete Chinese-English对照 Markdown reader.
```

When installing other skills, simply replace `nature-reader` in the example with the corresponding directory name, e.g., `nature-polishing`, `nature-writing`, `nature-reviewer`, `nature-response`, or `nature-figure`. Subsequent updates only require:

```bash
cd ~/ai-skills/nature-skills
git pull
```

As long as the wrapper still points to this stable clone path, there is no need to repeatedly copy skill files.

**Auto-Update (Optional)**

If you want Claude Code to automatically pull upstream updates every time a session starts, you can use `scripts/autoupdate-skills.sh` with a `SessionStart` hook.

This method **directly copies** skills into `~/.claude/skills/` (Claude Code will automatically discover this directory; skills are loaded directly by directory name), rather than using the wrapper approach above. Choose one of the two methods.

First keep a **dedicated** stable clone (only used for synchronizing skills; please do not make development commits inside it):

```bash
mkdir -p ~/ai-skills
git clone https://github.com/Yuan1z0825/nature-skills.git ~/ai-skills/nature-skills
```

First-time installation, copy skills into Claude Code's skill directory:

```bash
~/ai-skills/nature-skills/scripts/autoupdate-skills.sh --force
```

Then add a `SessionStart` hook in `~/.claude/settings.json` (if you already have `hooks`, merge this item in, do not overwrite the whole thing):

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "$HOME/ai-skills/nature-skills/scripts/autoupdate-skills.sh",
            "async": true,
            "timeout": 120
          }
        ]
      }
    ]
  }
}
```

`async: true` lets it run in the background without blocking startup. The script has built-in protections: by default it does not repeat network checks within 6 hours; if offline or pull fails it automatically skips (`exit 0`, never hangs the session); it only re-synchronizes when the upstream HEAD actually changes; and it refuses to force-forward on clones with uncommitted changes. The newly pulled version will take effect on the **next** session start (skills in the current session have already been loaded). Run logs are at `~/.local/state/nature-skills/autoupdate.log`.

Target directory and check frequency are both configurable:

```bash
# Default syncs to ~/.claude/skills; use --dest to point elsewhere, e.g., Codex:
~/ai-skills/nature-skills/scripts/autoupdate-skills.sh --dest ~/.codex/skills
# Check at most once per hour:
~/ai-skills/nature-skills/scripts/autoupdate-skills.sh --throttle 3600
```

### 5.3 Codex Installation Method

It is recommended to use the repository's built-in script to install or update Codex skills. The script synchronizes all top-level skill directories under `skills/`, and does a `diff` verification after copying; it will not overwrite other unrelated Codex skills.

```bash
git clone https://github.com/Yuan1z0825/nature-skills.git
cd nature-skills
scripts/update-codex-skills.sh --pull
```

If you have already cloned the repository:

```bash
cd nature-skills
scripts/update-codex-skills.sh --pull
```

Verify whether the current Codex installation matches this checkout:

```bash
scripts/update-codex-skills.sh --check
```

If you use this script for long-term updates and want to clean up old skill directories that have been deleted upstream:

```bash
scripts/update-codex-skills.sh --pull --prune
```

`--prune` will only delete directories previously recorded by this script but no longer contained in the current repository. It will not guess and delete old directories when there is no history record on the first run.

You can also give the repository link to Codex and let Codex execute the installation script. Recommended prompt:

```text
Please install Codex skills from this repository:
https://github.com/Yuan1z0825/nature-skills.git

Please clone the repository and run scripts/update-codex-skills.sh --pull.
After installation, run scripts/update-codex-skills.sh --check to verify.
Please keep the complete skill directories under skills/, do not only copy SKILL.md.
```

If installing only a single skill, please explicitly state the skill name:

```text
Only install nature-reader from this repository:
https://github.com/Yuan1z0825/nature-skills.git

If this skill requires shared files, please also install skills/nature-shared.
```

Key rule: preserve the complete directory structure. Please copy or reference the entire skill folder, not just `SKILL.md`, because many skills depend on `references/`, `static/`, `manifest.yaml`, scripts, assets, or shared files.

The MCP service for `nature-academic-search` also requires separate `PUBMED_EMAIL` configuration; optional providers such as Scopus / ScienceDirect require local credential configuration. Do not write API keys into repository files.

After installation, please start a new Codex session, then naturally describe the task, for example:

**Auto-Update (Optional)**

Codex supports a global `SessionStart` hook. After keeping a dedicated clone, you can check for updates every time you start or resume a Codex session, and sync the new version to `~/.codex/skills/`.

First create a dedicated clone and complete the first sync:

```bash
mkdir -p ~/.codex
git clone https://github.com/Yuan1z0825/nature-skills.git ~/.codex/.nature-skills-src
~/.codex/.nature-skills-src/scripts/autoupdate-skills.sh \
  --dest ~/.codex/skills --force
```

Then create or merge `~/.codex/hooks.json`:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup|resume",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash \"$HOME/.codex/.nature-skills-src/scripts/autoupdate-skills.sh\" --dest \"$HOME/.codex/skills\"",
            "timeout": 75,
            "statusMessage": "Checking Nature Skills updates"
          }
        ]
      }
    ]
  }
}
```

If `hooks.json` already has other hooks, please merge the `SessionStart` item, do not overwrite the whole thing. After first enabling or modifying the hook, run `/hooks` in Codex to check and trust it. Codex currently executes command hooks synchronously, so here we rely on the script's built-in 6-hour throttling, 60-second network protection, and automatic offline skip to avoid repeated networking on every session or blocking startup due to update failure.

Update logs are at `~/.local/state/nature-skills/autoupdate.log`. Newly pulled skills usually take full effect in the next session.

## 7. Contributing & Development

### 7.1 Shared Design Principles

All skills adhere to the following principles:

1. **Prioritize primary sources**: Rules are based on published Nature content, official journal guidelines, or explicit local sources, rather than vague aesthetic preferences.
2. **Explicit over implicit**: Every rule should explain its rationale, not just give assertions.
3. **Perceive sections and task context**: Academic writing, figures, citations, and responses all depend on context; different paper parts use different logic.
4. **Output-first**: Every skill should return directly usable artifacts, such as pastable text, `.svg`, `.pptx`, `.docx`, or specific suggestions.
5. **Extensible**: Each skill is self-contained in its own directory; adding new skills should not require modifying existing skills.

### 7.2 Repository Directory Structure

```text
skills/
├── nature-shared/              # Keep when skills reference ../nature-shared
├── nature-<topic>/
│   ├── README.md
│   ├── README_EN.md
│   ├── SKILL.md
│   ├── manifest.yaml     # Included for router-style skills
│   ├── static/           # Included for router-style skills
│   └── references/...
└── nature-proposal-writer/
    ├── README.md
    ├── README_EN.md
    ├── SKILL.md
    ├── scripts/...
    ├── templates/...
    └── references/...
```

### 7.3 New Skill Workflow

When adding skills to this repository, please follow this workflow:

**1. Create skill directory**

```text
skills/nature-<topic>/
```

**2. Add required files**

| File | Required | Purpose |
|------|----------|------|
| `SKILL.md` | Required | frontmatter (`name`, `description`) + rules + workflow; loaded by agent after trigger |
| `README.md` | Required | Human-facing Chinese description document |
| `README_EN.md` | Required | English description document accompanying the Chinese details page |
| `references/*.md` | Recommended for complex skills | Modular rule files, e.g., API, design theory, tutorials, chart types, etc. |

**3. Write Chinese and English READMEs**

Every new skill must provide both `README.md` and `README_EN.md`. The README is a human-facing skill entry page, not a repetition of `SKILL.md`, nor an installation manual. Its goal is to let users judge in 30 seconds: can this skill solve my problem, what do I need to give it, what will it produce, and where are its boundaries.

Basic rules:

- Chinese README and English README must mirror each other: same number of headings, same order, same information points; the English page should not be written as a separate independent template.
- Top is fixed as skill name, language switch link, and one positioning sentence.
- Use the following basic structure by default; only insert optional sections when truly needed.
- Do not repeat repository installation tutorials, author information, changelogs, development processes, complete file trees, or long internal implementation details in a single skill README.
- Complex rules, API parameters, long tutorials, script descriptions, and template indexes should go into `references/`, `static/`, `scripts/`, or `SKILL.md`; the README only keeps signposts.
- If the skill has visual assets, a small preview table can be placed; do not turn the README into a large image gallery or lengthy technical manual.

Chinese README basic structure:

```markdown
# `nature-<topic>` 技能

[English](README_EN.md)

One sentence explaining the skill's positioning, main task, and usage boundary.

## 适合用它做什么
## 典型请求
## 你需要提供
## 产出
## 边界
## 相关技能
```

English README must correspond to:

```markdown
# `nature-<topic>` Skill

[中文说明](README.md)

One sentence describing the skill's role, main task, and usage boundary.

## What To Use It For
## Typical Requests
## What You Need To Provide
## Outputs
## Boundaries
## Related Skills
```

Optional sections must be inserted in both languages synchronously, maintaining the same order. Common optional sections include:

| Chinese Section | English Section | Usage Scenario |
|----------|----------|------|
| `## 工作方式` | `## Workflow` | Need to explain core process or routing method |
| `## 运行和依赖` | `## Runtime and Dependencies` | Have scripts, MCP, API key, local configuration, or external dependencies |
| `## 示例预览` | `## Example Preview` | Have a small number of figures, screenshots, or visual assets worth showing |
| `## 内置参考` | `## Built-In References` | Need to point to `references/`, `assets/`, or demo |
| `## 方法来源` | `## Method Sources` | Writing, review, or analysis rules come from specific sources |
| `## 三种模式` | `## Three Modes` | Skill has clear compose/revise/hybrid modes, etc. |
| `## 与 ... 的关系` | `## Relationship With ...` | Easily confused with another skill, need to explain division of labor |

Before submitting, at least do these README checks:

```bash
python scripts/validate-readmes.py
git diff --check
for d in skills/nature-*; do
  [ -f "$d/README.md" ] && [ -f "$d/README_EN.md" ] || continue
  rg -q '^\[English\]\(README_EN\.md\)$' "$d/README.md"
  rg -q '^\[中文说明\]\(README\.md\)$' "$d/README_EN.md"
  test "$(rg -c '^## ' "$d/README.md")" = "$(rg -c '^## ' "$d/README_EN.md")"
done
```

**4. Record usage tutorial**

When submitting a PR, please also record a short usage tutorial explaining what problem this skill solves, how to trigger it, what input is needed, and what results it will produce. You can attach video, screen recording links, or publicly accessible tutorial addresses in the PR description.

**5. Configure `SKILL.md` frontmatter**

```yaml
---
name: nature-<topic>
description: >-
  In one sentence, explain what this skill does, when to trigger it, main output format, and core usage scenario.
---
```

**6. Update skill index**

Add a row to the [Skill Index](#6-skill-index) table above:

```markdown
| [`nature-<topic>`](skills/nature-<topic>/README.md) | Draft / Stable | One-sentence purpose | Trigger words | [Details](skills/nature-<topic>/README.md) |
```

**7. Set status label**

| Label | Meaning |
|-------|------|
| `Draft` | Rules defined, but not yet tested on real cases |
| `Beta` | Tested on examples, may still have boundary issues |
| `Stable` | Validated on real academic content, rules relatively stable |
